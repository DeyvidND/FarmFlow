# ФермериБГ — Платформа (super-admin console)

## Design Context

### Users
Solo platform operator (the founder) running the ФермериБГ / FarmFlow SaaS for
Bulgarian farm-shops. A power user, on it daily-to-weekly, almost always on
desktop. Jobs to be done: onboard and manage **tenants** (farm brands),
manage the **producers** that sit under each brand, control access /
billing / premium, watch platform health + problems, impersonate a tenant to
fix things, and run delivery ops. This is a command centre, not a
consumer-facing app — density and speed beat hand-holding.

Note: the *tenant-facing* farmer panel (`client/`) is a separate app tuned for
non-digital elder farmers. This console is the opposite audience. Do not import
that app's elder-first constraints here — but DO share its visual language.

### Brand Personality
Earthy, grounded, operator-grade. Bulgarian agricultural roots, calm competence,
"платформена командна зала". Three words: **earthy · grounded · dependable**.
Emotional goal: the operator feels in control and oriented, never lost.

### Aesthetic Direction
Extend the EXISTING FarmFlow design system — do not reinvent it. This is a
shipped product with an established brand.
- **Palette**: warm earthy — cream bg `#F3EEE2`, white surfaces, olive-green
  primary `#2C5530` (`--ff-green-*`), amber accent (`--ff-amber-*`), warm ink
  `#26241D`. Neutrals are already tinted warm. Keep the 60/30/10 discipline.
- **Type**: Bitter (slab-serif, `--font-bitter`) for display/headings +
  Commissioner (humanist sans, `--font-commissioner`) for body. `ff-fig` (Bitter
  + tabular figures) for money/stats. NO new fonts.
- **Shape**: 12px radius, soft `ff-sm/md/lg` shadows, 1px borders.
- **Theme**: LIGHT only. Operator works in daylight over data-dense screens.
  (`darkMode:['class']` is configured but has no tokens — dead config.)
- **Anti-references**: generic dark SaaS dashboards, cyan-on-dark, purple→blue
  gradients, glassmorphism-everywhere, gradient text, KPI-tiles-with-gradient,
  and coloured left-stripe accent cards (already present in insights — remove).

### Design Principles
1. **One brand, one home.** Everything hangs off the farm-brand it belongs to.
   Producers are shown nested under their tenant, never as an orphaned flat list.
2. **A navigation spine, not a top rail.** Persistent left sidebar with grouped
   sections (mirrors the farmer app's shell), replacing the overflow-scroll
   top-bar that hides items.
3. **Legible data first.** AA contrast everywhere (retire muted text < 4.5:1 for
   meaningful content), tabular figures for numbers, clear table hierarchy.
4. **On-brand, tokenised.** Reuse `ff-*` tokens; tokenise the stray hex
   (`#3457B1` demo-blue etc.). No one-off colours.
5. **Calm density.** Power-user information density with rhythm — varied spacing,
   grouped meaning, no wall-to-wall uniform cards or noise.
