# Delivery address street/note split — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep the customer's street and their block/entrance detail (бл./вх./ет./ап.) in separate fields all the way to a new `orders.delivery_note` column, so the courier detail never pollutes server-side geocoding for local delivery.

**Architecture:** chaika's checkout already has two inputs (`addrInput` street + `addrDetails` block/entrance). Today `composeAddress()` glues them into one `deliveryAddress` string that the backend geocodes — the block/entrance noise makes Google reject or mis-snap the result. Fix: for the **local `address`** delivery branch only, send the street in `deliveryAddress` and the detail in a new `deliveryNote` field. `econt_address` is left untouched (Econt's waybill needs the full string and is never geocoded). The backend stores `deliveryNote`, the geocode query is automatically clean, and the farmer's order panel + route stop list display the note for the driver.

**Tech Stack:** Drizzle ORM + drizzle-kit (Postgres), NestJS + class-validator (backend), Next.js/React (admin), Astro + TS (chaika storefront), Jest (server tests).

**Spec:** `docs/superpowers/specs/2026-06-21-delivery-address-street-note-split-design.md`
